Introduction

This extension lets you attach XSL files (whether through the "file:///" protocol, through a local file (e.g., "C:\..."), through a URL (i.e., "http://..."), or through a manually typed XSL file) to the currently displayed XML (or HTML) file.

Operating System requirements

The code is known to work on Windows. If anyone can help to test to make sure it works correctly on other systems, please contact me (brettz9 A-T the well-known site Yahoo.com).

Known issues:

  1. XSLT 2.0's <xsl:result-document> element is not supported.
  2. Built-in processor will not work for XSL files with <xsl:include> or <xsl:import> statements.
  3. Due to changes in FF3 (at least b2), one can not paste directly (e.g., a URL) into the file browser textbox, but it is possible if done through the file dialog (at least on Windows).

Choice of XSLT processor

You can currently only use Firefox's built-in XSLT processor.

Performing a transformation via the XSL Dialog

  1. Choose Get XSL Results from the Tools menu to open the XSL Results window.

    If you wish to perform a transformation on the currently loaded window/tab in Firefox, be sure to wait until the document has fully loaded before activating the XSL Results window, or otherwise you will not be able to transform the full text of that document and you will mostly likely get well-formedness errors. However, you are always free to edit the source XML document after loading the XSL Results window.

    Be aware that due to either a bug or a feature in Firefox, in serialization, HTML gets converted (normalized) to upper case, so be aware that XSL Results will convert HTML tag names all back into lower case so that your transformations can use the more standard lower case tag names (whether or not the original document did or not). True XML files, including XHTML (served as application/xhtml+xml) will be left alone: http://www.mozilla.org/docs/web-developer/faq.html#xhtmldiff).

    The extension will also attempt to fix the resulting code so that specially (or poorly) crafted HTML will become well-formed XML when converted into Firefox's DOM representation. For example, comments of the form <!--- which have an extra hyphen, while still preserved as such by Firefox in its inner representation of such documents, will be converted by this extension into the well-formed style <!-- so as to allow the XSLT engine to be able to transform such documents without running into errors.

    There are also recommended options in preferences (turned on by default) which may strip the DTD (recommended as explained in the interface). (DTD/schema validation of source documents is turned off for BDBXML.) If the automated options are off, you will still have a chance to click a button to strip the DTDs (if a DTD is present). HTML DTD's can often cause problems as SGML-style DTD's differ from XML ones, so these are stripped by default. Automatic DTD stripping is on by default for XML as well as for HTML because it is redundant for true XHTML (Firefox relies on built-in entities) and since other XML documents dependent on the external DTD will currently trigger a parsing error in Firefox anyhow (in such a case, you can manually paste in the source of the current document). Be aware that if you paste in your own XML, the DTD will not be automatically stripped (in such a case, the DTD IS needed for entities and default attributes if you have any, but if you don't have them, it is unnecessary, slower, and taxes the server hosting the DTD).

  2. Choose whether to also have output produced in a new window and choose what file extension you want the resulting file to possess. If this is made blank, an <xsl:output> element may be used to determine the extension type.

  3. Choose or add your XSL file.

    You have the choice of loading an external file for the XSL stylesheet or to paste or type the text in directly.

  4. Hit 'Transform' on the dialog. Be aware that depending on the complexity of your transformation, on whether you are loading files over the network, and on whether you left the checkbox on to save the current file, it may take a while to process, but eventually the contents should appear in the XML Output textbox and if the option is selected, a new window should open with the results of your transformation. If the window is blank, consider that you may need to view the textbox contents or "Page source" of the document.

    Note that any time you hit 'Transform', the window and splitter positions will be saved for next time. The same behavior occurs by hitting 'Ok', but that will not perform a transformation. Any time most of the other fields are changed, they will be retained in memory the next time the window is opened.

Obtaining the resultant code of an already transformed document loaded in the current FF tab/window

  • There is also a context menu item available to immediately grab the transformation results of the current XML document (using Firefox's built-in XSLT processor) with an already-attached stylesheet and places the results in a textbox (achieves a similar effect as the built-in (yet-not-so-easily-discoverable) Firefox behavior of selecting-all on an XML document and choosing "View selection source").
